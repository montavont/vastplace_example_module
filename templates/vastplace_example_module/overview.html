CDF Graph

This graph is generated using the views.metal_cdf function. It looks for all the traces of type vastplace_example, and all the sensor they contain. It computes the CDF of all the values of the sensors of type 'metal' it discovered

Bar Graph

This graph is generated using the views.metal_bar function. It looks for all the traces of type vastplace_example, and all the sensor they contain. It computes the bar graph of values depending on wether they are above or below 30000

Step Graph

This graph is generated using the views.metal_step function. It looks for all the traces of type vastplace_example, and all the sensor they contain. It then represent the value of the sensor depending on its timestamp

Rest API

This item is the response to a rest request. It looks for all the traces of type vastplace_example, and all the sensor they contain. It stores the number of element in a dictionnary depending on wether they are above or below 30000

Tile map

This map is generated by calling the views.tile_map function. It looks for all the traces of type vastplace_example, and all the sensor they contain. It then locates the sensors of type 'metal' using the GPS measurements in the trace and locates them in tiles. In each tile, the average sensor value is calculated, and displayed as letters and colorcode

Cell map

This map is generated by calling the views.cell_map function. It looks for all the traces of type vastplace_example, and all the sensor they contain. It then attributes the sensors of type 'metal' to the closest geographic feature in the area, using openstreetmap data. In allocates the measurements to cells, and creates a colorscale depending on wether the interpolated value of each cell is closer from 6000 or 60000
